# Commit Convention

以后提交信息建议使用以下格式：

```text
type: short description
```

常用类型：

- `feat:` 新增功能或内容
- `fix:` 修复问题
- `style:` 视觉、样式、格式调整，不改变逻辑
- `docs:` 文档更新
- `chore:` 工程配置、依赖、部署相关维护

示例：

```text
docs: update README contact info
style: adjust project card spacing
fix: correct image path on work detail page
chore: update Vercel config
```
